# 1. Класс Display
Конструктор
```python
Display(spi, width, height, dc, cs=None, rst=None, bl=None, backlight_active_high=True, rotation=0, buffer=None)
```

### Параметры:
- spi – объект machine.SPI (уже инициализированный)
- width, height – размеры дисплея в пикселях
- dc – пин Data/Command (machine.Pin)
- cs – пин Chip Select (machine.Pin, опционально, может быть None)
- rst – пин Reset (machine.Pin, опционально)
- bl – пин или PWM для подсветки (опционально)
- bgr – bool, настройки цветового порядка BGR/RGB
- backlight_active_high – bool, уровень включения подсветки (по умолчанию True)
- rotation – поворот: 0, 90, 180, 270 (по умолчанию 0)
- buffer – опциональный bytearray размером width * height * 2. Если None – буфер выделяется внутри драйвера.

### Что делает конструктор (в C-реализации):
- Выделяет память через malloc (или использует переданный bytearray).
- Создаёт объект bytearray через mp_obj_new_bytearray_by_ref.
- Сохраняет spi, пины, размеры, rotation.
- Не выполняет инициализацию дисплея (ни сброса, ни команд).

### Свойства (только чтение)
- buffer – bytearray кадрового буфера (можно менять содержимое, но нельзя переприсвоить).
- width, height – размеры.
- rotation – текущий поворот (можно изменять: при записи обновляется регистр MADCTL).

### Методы
- cmd(command) – отправить один байт команды.
- cmd_data(command, data) – отправить команду и данные (data – bytes или bytearray).
- show() – отправить весь буфер на дисплей.
- update_rect(x, y, w, h) – отправить прямоугольную область (с учётом rotation, клиппинг).
- set_backlight(percent) – установить яркость (0–100). Для PWM – скважность, для Pin – вкл/выкл.
- reset() – импульс сброса на пине rst (10 мс низкий, 10 мс высокий), если rst задан.

# 2. Класс ST7735 (наследник Display)
## Конструктор
```python
ST7735(spi, width, height, dc, cs=None, rst=None, bl=None, backlight_active_high=True, rotation=0, buffer=None)
```

## Параметры – такие же, как у Display.
## Что делает дополнительно:
- Вызывает конструктор Display.
- Выполняет аппаратный сброс (если есть rst).
- Отправляет последовательность инициализации ST7735: выход из сна, установка формата RGB565, настройка окна CASET/RASET с учётом width/height, настройка MADCTL согласно rotation, включение дисплея, все необходимые задержки.

Все остальные методы и свойства наследуются от Display без изменений.

# 3. Пример использования
```python
from machine import Pin, SPI
from spi_displays import ST7735
import framebuf

spi = SPI(1, baudrate=20_000_000, sck=Pin(10), mosi=Pin(11), miso=None)
tft = ST7735(
spi=spi,
width=128,
height=160,
dc=Pin(8),
cs=Pin(5),
rst=Pin(9),
bl=Pin(13),
rotation=0
)

fb = framebuf.FrameBuffer(tft.buffer, tft.width, tft.height, framebuf.RGB565)
tft.display = fb # опционально, для удобства

tft.display.fill(0x0000)
tft.display.text("Hello", 10, 10, 0xFFFF)
tft.show()
tft.set_backlight(100)
```

### Частичное обновление:
```python
tft.display.fill_rect(0, 0, 50, 50, 0xF800)
tft.update_rect(0, 0, 50, 50)
```

### Использование своего буфера:
```python
my_buffer = bytearray(128 * 160 * 2)
tft = ST7735(..., buffer=my_buffer)
```

### Ручная инициализация через Display (для отладки):
```python
disp = Display(spi=spi, width=128, height=160, dc=Pin(8), ...)
disp.reset()
disp.cmd(0x01)
time.sleep_ms(150)
disp.cmd_data(0x3A, b'\x55')

... остальные команды
fb = framebuf.FrameBuffer(disp.buffer, disp.width, disp.height, framebuf.RGB565)
fb.fill(0xFFFF)
disp.show()
```

# 4. Требования к реализации на C
- extmod/machine_spi.h не входит в стандартные пути включения для внешних C-модулей. ( fatal error: extmod/machine_spi.h: No such file or directory)
- Буфер: выделять через malloc, если buffer=None; иначе проверять размер переданного bytearray.
- Создание bytearray: mp_obj_new_bytearray_by_ref.
- Освобождение памяти: в финализаторе объекта (del), если буфер выделен внутри, вызвать free().
- Свойства: геттеры для buffer, width, height; сеттер для rotation (обновляет MADCTL).
- update_rect: преобразование координат согласно rotation, вычисление смещения в буфере, отправка через SPI построчно или одним блоком.
- Обработка ошибок: при сбое SPI – mp_raise_RuntimeError.
- Сборка: внешний C-модуль (USER_C_MODULES), micropython.cmake, макрос MICROPY_PY_SPI_DISPLAYS=1.
- Платформы: ESP32, ESP32-C3, ESP32-S3, RP2040.
